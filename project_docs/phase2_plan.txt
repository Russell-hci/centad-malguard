# SECTION 3 — Malware Image Pipeline

This is where your methodology becomes real.

Your literature review found that simple grayscale mapping often performs as well as more complex representations.

Therefore:

# Recommended Initial Approach

Use:

* existing malware images from MalImg

DO NOT:

* implement Hilbert curves yet
* implement custom byte visualizations yet

Keep Phase 2 simple.

---

# Image Standardization

Convert all images to:

```python
224 × 224
```

Reason:

* MobileNet
* EfficientNet
* ShuffleNet

all support this naturally. ([PyTorch Docs][2])

---

# Data Normalization

Use ImageNet normalization:

```python
mean=[0.485, 0.456, 0.406]
std=[0.229, 0.224, 0.225]
```

because:

* pretrained models expect it.

---

# Data Augmentation

ONLY basic augmentation initially:

```python
RandomHorizontalFlip
RandomRotation
RandomResizedCrop
```

Do not overcomplicate.

---

# Use Transfer Learning

Do NOT train from scratch.

Use pretrained ImageNet weights.

PyTorch already provides pretrained versions. ([PyTorch Docs][2])

---

# Recommended Training Strategy

### Stage 1

Freeze backbone:

```python
for param in model.features.parameters():
    param.requires_grad = False
```

Train classifier head only.

---

### Stage 2

Unfreeze final layers.

Fine-tune.

This improves stability.
